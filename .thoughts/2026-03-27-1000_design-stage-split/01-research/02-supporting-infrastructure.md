## Summary

The supporting infrastructure around the design stage includes the stage creator (generates README.md/PHASES.md), the implement reviewer (scoped exclusively to 04-implement), the manifest (holds rdpi bundle version `1.0.4`), workflow instructions (define stage structure and status conventions), and per-stage skills (each defining their own README.md frontmatter schema). A `pipeline-version` frontmatter field exists only in the research-reviewer agent template — it is not generated by the stage creator, not defined in any stage skill's Output Conventions, and not propagated from the manifest. The implement reviewer has zero design-awareness capabilities and is referenced only in the 04-implement context.

## Findings

### 1. Stage Creator Agent — README.md Generation

- **Location**: `@/templates/rdpi/agents/rdpi-stage-creator.agent.md:68-83`
- **What it does**: Generates the initial `README.md` for each stage with a YAML frontmatter template:
  ```yaml
  title: "<Stage Title>: <Feature Name>"
  date: <YYYY-MM-DD>
  status: Inprogress
  feature: "<brief feature description>"
  ```
- **Frontmatter adaptation rule** (line 77): "Adapt frontmatter per stage Output Conventions from the stage skill. Always include cross-reference fields defined there (e.g., `research` for 02-design, `research` + `design` for 03-plan, `plan` for 04-implement)."
- **No `pipeline-version` field**: The stage creator's README.md template does not include a `pipeline-version` field. It relies entirely on what the stage skill's Output Conventions specify.
- **Key dependency**: Reads the stage skill file (`rdpi-<stage-identifier>`) at Step 2 (line 49) to determine frontmatter fields and output structure.
- **Patterns**: The stage creator is a meta-agent — it plans work but does not execute it. It creates exactly two files: `README.md` and `PHASES.md`.

### 2. Implement Reviewer Agent

- **Location**: `@/templates/rdpi/agents/rdpi-implement-reviewer.agent.md:1-115`
- **What it does**: Reviews implementation for plan compliance, code quality, and produces an implementation record README.md. Two tasks: quality review checklist + implementation record.
- **Current scope**: Exclusively tied to `04-implement` stage:
  - Reads `03-plan/README.md` and all plan phase files (line 30)
  - Reads `04-implement/verification-*.md` (line 31)
  - Writes `04-implement/README.md` (line 62)
- **References outside 04-implement**: None. All 9 occurrences of `rdpi-implement-reviewer` across the template files are:
  - `@/templates/manifest.json:60-61` — bundle item definition
  - `@/templates/rdpi/instructions/thoughts-workflow.instructions.md:80` — role listing under "04-Implement"
  - `@/templates/rdpi/agents/rdpi-approve.agent.md:11` — generic list of stage reviewers
  - `@/templates/rdpi/agents/rdpi-stage-creator.agent.md:183` — redraft mode reviewer role mapping ("for implement")
  - `@/templates/rdpi/skills/rdpi-04-implement/SKILL.md:16,26,42` — role table and phase structure
  - `@/templates/rdpi/agents/rdpi-implement-reviewer.agent.md:2` — self-reference
- **Design-awareness capabilities**: None. The agent's checklist (line 41-48) checks plan compliance, verification reports, out-of-scope files, code patterns, barrel exports, TypeScript strictness, documentation proportionality, and security. No mention of design documents, architecture, or design traceability.
- **Frontmatter output**: `(title, date, status, feature, plan)` — references plan only, not design or research.

### 3. Manifest — Bundle Version

- **Location**: `@/templates/manifest.json:1-95`
- **rdpi bundle version**: `"version": "1.0.4"` at `@/templates/manifest.json:18`
- **Format**: Semantic versioning string within the `bundles.rdpi` object.
- **CLI injection mechanism**: During installation, the CLI injects `astp-version` (among other `astp-*` fields) into each installed file's YAML frontmatter (`@/templates/README.md:82-93`). This is the bundle version at install time.
- **No runtime propagation to .thoughts/**: The `astp-version` field is injected into installed template files (agents, skills, instructions), NOT into `.thoughts/` stage README.md files. There is no mechanism to propagate the manifest version to stage README.md frontmatter.

### 4. Workflow Instructions

- **Location**: `@/templates/rdpi/instructions/thoughts-workflow.instructions.md:1-81`
- **README.md conventions** (lines 23-32):
  - Front matter required on each file
  - Status field values: `Inprogress`, `Draft`, `Review`, `Approved`, `Redraft`
  - Cross-references between documents via relative links
  - File paths with `@/` alias
- **No frontmatter schema definition**: The workflow instructions define status values and cross-reference conventions but do NOT define a complete README.md frontmatter schema. Each stage skill defines its own frontmatter fields in "Output Conventions".
- **No `pipeline-version` mention**: The workflow instructions contain no reference to pipeline version, manifest version, or any version field in README.md frontmatter.
- **Stage listing** (lines 55-58): Four stages defined (01-research through 04-implement).
- **Role assignments** (lines 60-81): `rdpi-implement-reviewer` is listed exclusively under "04-Implement".

- **README.md frontmatter fields**: `(title, date, status, feature)`
- **No cross-reference fields** (first stage — nothing precedes it)
- **No `pipeline-version` field**

#### 02-Design (`@/templates/rdpi/skills/rdpi-02-design/SKILL.md:74`)
- **README.md frontmatter fields**: `(title, date, status, feature, research)`
- **Cross-reference**: `research` points to `../01-research/README.md`
- **No `pipeline-version` field**

#### 03-Plan (`@/templates/rdpi/skills/rdpi-03-plan/SKILL.md:78`)
- **README.md frontmatter fields**: `(title, date, status, feature, research, design)`
- **Cross-references**: `research` and `design`
- **No `pipeline-version` field**

#### 04-Implement (`@/templates/rdpi/skills/rdpi-04-implement/SKILL.md:92`)
- **README.md frontmatter fields**: `(title, date, status, feature, plan)`
- **Cross-reference**: `plan`
- **No `pipeline-version` field**

**Pattern**: No stage skill includes `pipeline-version` in its Output Conventions frontmatter definition. The only occurrence of `pipeline-version` is in the `rdpi-research-reviewer` agent template (`@/templates/rdpi/agents/rdpi-research-reviewer.agent.md:65`) with the instruction `"<preserve from existing README.md>"` — implying it expects the field to already exist when the reviewer runs, but nothing currently generates it.

### 6. 04-Implement Multi-Agent Pattern

- **Location**: `@/templates/rdpi/skills/rdpi-04-implement/SKILL.md:19-50`
- **Pattern**: Each plan phase becomes a code → test pair (`rdpi-codder` then `rdpi-tester`), followed by a single `rdpi-implement-reviewer` at the end.
- **Retry loop**: If tester reports failures, orchestrator re-spawns the coder with verification report context, then re-runs tester. Repeats until pass or retry limit exhausted.
- **Parallelism**: Independent plan phases produce independent implement phase pairs that run in parallel (lines 28-42).
- **Scaling rule**: Never exceed `2x (number of plan phases) + 1` total phases.
