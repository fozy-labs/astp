## Impact on Existing Installed Pipelines

**Re-installation required**: Yes. Users must run `astp update` to get the updated template files. There is no automatic migration.

**Migration path**:
1. Run `astp update` — replaces installed template files with new versions, re-injects `astp-version`.
2. Existing `.thoughts/` stages already in progress are **not affected** — they use the PHASES.md already generated by the old skill. Only new stages created after the update use the new phase structure.
3. No breaking changes to existing stage outputs — the new design skill adds capabilities (per-document tiers, correction log) but does not invalidate existing design documents.

**Backward compatibility**: Stages created with the old skill (grouped tiers, no correction log) remain valid. The design reviewer's Pass 2 handles the "no corrections" case gracefully.
